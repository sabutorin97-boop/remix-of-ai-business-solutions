# Перенос проекта на Timeweb Cloud

## Что переносим
1. Сайт-визитка студии (TanStack Start, текущий проект)
2. Telegram-бот студии (AI-ассистент на KIE.ai)
3. Telegram-бот AI-Лина (нутрициолог)

## Выбор тарифа Timeweb Cloud

**Рекомендация: Cloud Server "Disel" или аналог**
- 2 vCPU / 2 GB RAM / 40 GB NVMe
- Ubuntu 22.04 LTS
- ~400–600 ₽/мес
- Локация: Москва (низкий пинг для РФ-аудитории)

Этого хватит на сайт + 2 бота с запасом. Если ботов станет больше или появится тяжёлая обработка фото — апгрейд до 4 GB RAM.

Альтернатива — **Timeweb Apps** (PaaS, деплой из Git), но он не поддерживает Cloudflare Workers рантайм, поэтому проект придётся переводить на Node-адаптер. Поэтому ниже план под классический VPS.

## Шаги переноса

### 1. Подготовка проекта под Node.js рантайм
Сейчас проект собран под Cloudflare Workers (`wrangler.jsonc`, `src/server.ts`). Для VPS нужно:
- Переключить TanStack Start на Node-сервер (адаптер node-server)
- Обновить `vite.config.ts` — убрать Workers-таргет
- Проверить серверные функции (`*.functions.ts`) — все совместимы с Node, специфики Workers в проекте нет
- Собрать прод-бандл (`bun run build` → `dist/`)

### 2. Настройка сервера Timeweb
- Создать Cloud Server, получить root-доступ по SSH
- Базовая настройка: пользователь, ufw (22, 80, 443), fail2ban
- Установить Node.js 20 LTS, Bun, Nginx, certbot, PM2, git

### 3. Деплой сайта
- Клонировать репозиторий в `/var/www/ai-profigrup`
- `bun install && bun run build`
- Запуск через PM2: `pm2 start dist/server/index.mjs --name site`
- Nginx как reverse proxy на 127.0.0.1:3000
- SSL через Let's Encrypt для основного домена

### 4. Деплой Telegram-ботов
Каждый бот — отдельный Node-процесс под PM2:
- `pm2 start bot-studio.js --name tg-studio`
- `pm2 start bot-ailana.js --name tg-ailana`
- Webhook-режим: Telegram → Nginx (`/webhook/<token>`) → бот на отдельном порту
- Либо long-polling, если ботов мало и нагрузка низкая (проще, не нужен публичный URL для каждого)
- Секреты (BOT_TOKEN, KIE.ai API ключ) — в `.env` файлах с правами 600

### 5. DNS и домены
- В панели регистратора домена прописать A-запись на IP сервера Timeweb
- Для ботов отдельные субдомены (bot.example.ru, ailana.example.ru) или общие пути на основном домене

### 6. Автозапуск и мониторинг
- `pm2 startup` + `pm2 save` — автозапуск после ребута
- Логи: `pm2 logs`
- Опционально: Uptime Kuma или Timeweb-мониторинг для алертов

### 7. Бэкапы
- Включить ежедневные снапшоты диска в панели Timeweb (~50 ₽/мес)
- Git как основной источник кода

## Технические нюансы

- **Workers → Node переход**: серверные функции TanStack Start работают и там, и там, но текущий `src/server.ts` написан под Workers fetch-handler. Нужен Node-адаптер.
- **Окружение**: все `process.env.*` остаются как есть, просто читаются из `.env` через PM2 или systemd.
- **Боты в одном проекте или отдельных репо**: рекомендую отдельные репозитории/папки — у них свой жизненный цикл и зависимости.

## Что потребуется от вас перед стартом
- Доступ к домену (где куплен, можем ли менять DNS)
- Репозитории кода ботов (или их код — сейчас в проекте только сайт)
- Решение по тарифу Timeweb (2 GB или сразу 4 GB)
- Готовы ли отказаться от Cloudflare Workers (с VPS теряется бесплатный CDN — можно компенсировать Cloudflare в режиме DNS-прокси перед Timeweb)
